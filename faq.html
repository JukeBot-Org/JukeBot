üéß Will Spotify be supported?

This is actually the reason I began writing JukeBot: to have a music bot that can play music from a Spotify URL. However, implementing this is trickier than I thought. Each instance of JukeBot would require its owner to register a Spotify application as well in order for the bot to access the Spotify API. Given that this is a hurdle that can be overcome (albeit with a lot of work), it will probably be implemented way down the line.

‚òÅÔ∏è What about SoundCloud?

For Soundcloud, the issue is similar but much less possible; SoundCloud's API is currently not accepting requests for new API keys and has no public roadmap for when requests will be open once more.
Again, this is an issue that can be solved, but requires a lot of work for the user, which runs against JukeBot's accessibility ethos.

üìÅ Can you host the bot for me?

Jukebot was built after the take-downs of multiple other Discord audio streaming bots. I lamented the fact that these bots were closed-source, meaning I couldn't run my own local instance for personal use, which left my friends and I without a way to listen to music together.

I made the decision to make JukeBot self-hosted for two reasons: firstly, controlling your own instance of JukeBot lets you keep the bot up in the event that the source code goes away at some point and preventing Jukebot from falling apart once a hypotetical "one bot" would be taken down.

Secondly, it was easier to program that way. Work smarter, not harder, hey?
